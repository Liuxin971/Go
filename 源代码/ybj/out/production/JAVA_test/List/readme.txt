数组与集合
    都可以存储多个对象，对位作为整体

  1、数组的缺点
          数组长度必须初始化,且固定不变
          数组采用连续存储空间，删除和添加效率低下
          数组无法保存映射关系
          数组缺乏封装，操作繁琐
  2、集合的引入位于java.util
           Collection __ Map
       Collection 不唯一 无序
       List     不唯一 有序
       Set       唯一  无序
       Map       键值对 key value
  List <有序不唯一>
  ArrayList
          内存中分配连续空间，实现可变的数组
          索引查遍历比较高
          添加删除,按照内容降低查询效率
  LinkedList
          采用双向链表
          缺点：遍历随机访问地下
          优点：(先低效率查询 )插入删除元素效率低下
  二泛型
          不安全
          简单
          //
  二叉平衡树
  二叉排序数
  2^n=sum n种方式
  list sum/2 有序 不唯一
  hashset  底层结构hash 速度快 无序 唯一
  linkedhashset 哈希表+链表  有序 唯一 速度快 添加顺序
  treeSet 底层结构是 红黑树 速度快 有序(大小顺序) 唯一
